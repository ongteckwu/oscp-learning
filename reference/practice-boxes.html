Practice Box Map

Every skill lesson, tied to concrete machines from HTB, Proving Grounds and PEN-200. Expand a lesson to see its boxes — each tagged by vector and difficulty and marked Core (do these for full coverage) or Optional, with a checkbox to track what you've owned. Work them attempt-first (Lesson 02): enumerate yourself, spend a struggle budget, take one nudge, then review.

Read before you queue a box
  • TJ Null's list is the source of truth. This map is curated from it (+ 0xdf's OffSec mapping) for teaching; the live sheet is authoritative on what's currently in-scope. Ask me to verify any box.
  • Most HTB boxes here are retired → they need an HTB VIP subscription (the retired-machines list). For OSCP prep that's the standard move.
  • "PEN-200" boxes aren't individually public anymore. OffSec replaced the named lab with six Challenge Labs — scenario (Medtech, Relia, Skylark) and exam-like (OSCP A, B, C). Proving Grounds Practice is the closest public stand-in to exam style.
  • Don't spoil the chain. Pick boxes you haven't seen a write-up for; only watch IppSec/0xdf in review mode after you finish or exhaust yourself. Each HTB box's 0xdf ↗ badge links straight to its walkthrough — strictly for that after-the-fact review.

15 · Writing the Report

No new box — pick one you've already rooted and write it up to exam standard (every command, every screenshot, proof files). Run it through the /box-notes skill.

16 · Exam Strategy

No new box — this is decision-making, not a new technique. Rehearse the time-boxing and "when to move on" rules on the Lesson 17 mock-exam sets.

How to drive this map Interleave OS and box type — don't binge five Linux privesc boxes in a row (that builds false fluency; see Lesson 02). After each box, run it through /box-notes so the transferable technique lands in your growing box-techniques cheat sheet. When you can pick the right box for a weakness without this map, you've internalised it.
